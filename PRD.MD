# EPC 项目合同风险智能评估系统 - 产品需求文档 (PRD)

## 1. 项目概述

本项目旨在开发一款基于大语言模型（LLM）与多智能体协作（MAS）的 EPC 合同风险评估系统。系统通过模拟人类专家组的协审流程，结合检索增强生成（RAG）技术与多目标优化算法（NSGA-III），为承包商提供具备法律依据、可量化的风险决策支撑。

## 2. 多智能体系统 (MAS) 职责定义

系统核心由“1 个初筛门卫 + 6 个专业审计专家”构成：

| 智能体名称                 | 核心职责                               | 挂载指标 (AHP 权重数据参考) |
| :------------------------- | :------------------------------------- | :-------------------------- |
| **初筛门卫 (DeepSeek-R1)** | 逻辑拆解、敏感条款识别、分类路由       | N/A (总控中枢)              |
| **经济风险专家 (Qwen3)**   | 支付规则、调价机制、汇率风险审计       | **支付风险 (0.4911)**       |
| **法律风险专家 (Qwen3)**   | 争议解决、合规性校验、知识产权归属     | **争议解决+合规 (0.2975)**  |
| **管理风险专家 (Qwen3)**   | 工程变更、索赔程序、分包管理漏洞       | **索赔+分包+变更 (0.1728)** |
| **技术风险专家 (Qwen3)**   | 设计范围界定、工期责任、性能验收标准   | **设计+工期 (0.0381)**      |
| **社会风险专家 (Qwen3)**   | 征地拆迁、外部环境协调、农民工工资保障 | **社交协调 (<0.01)**        |
| **自然风险专家 (Qwen3)**   | 不可抗力条款、极端天气免责机制审计     | **自然灾害 (<0.01)**        |

## 3. RAG 检索增强引擎

- **知识库集成**：引入《民法典》、FIDIC 合同条款、EPC 行业标准及过往 500+ 典型纠纷案例。
- **证据链逻辑**：系统必须输出“风险条款 -> 推导后果 -> 修订建议 -> 法律依据”四位一体的审计结论。

## 4. 博弈聚合与多目标寻优 (NSGA-III Implementation)

系统采用 **NSGA-III** 算法对比多个 Agent 的风险判断，寻找帕托最优（Pareto Optimal）的共识意见。每个维度的最终风险分值均由以下三个数学目标共同决定：

### 4.1 核心优化目标定义

1.  **共识水平 ($A_{TOT}$ 最大化)**
    - **实现逻辑**：计算聚合后的概率分布 (PDF) 与各专家 Agent 原始分布的“总重叠面积”。
    - **目标**：确保最终结论能最大限度地代表所有专家的集体意见，减少离群值的干扰。

2.  **置信水平 ($\sigma^2$ 最小化)**
    - **实现逻辑**：衡量聚合分布的“集中程度”（Sharpness），通过计算 PDF 的方差 $\sigma^2$ 实现。
    - **目标**：方差越小，说明专家组对该风险的判断越一致、越明确。

3.  **公平效用水平 ($U$ 最大化)**
    - **实现逻辑**：基于行为经济学 **Fehr-Schmidt (F-S) 模型**，动态计算各 Agent 间的嫉妒效用损失与同情效用增益，保护“少数派专家的正确意见”。
    - **目标**：防止聚合结果被某种单一模型或高权重指标完全霸权化。

### 4.2 算法实现路径

- **底层架构**：采用 `pymoo` 框架实装 NSGA-III 寻优逻辑。
- **归一化处理**：使用带 5% 缓冲区间（Buffer）的 Min-Max 归一化，确保帕累托前沿不因异常值断裂。

## 5. 前端交互与可视化

- **雷达分析视图**：实时展示法律、财务、进度、权责、证据 5 个维度的聚合评分。
- **响应式 UI**：采用现代设计语言，整合 Framer Motion 实现风险审计流转动画。

## 6. 技术架构：Next.js 前后端一体化

考虑到开发效率与 UI 交互体验，本系统采用 **Next.js + TypeScript** 的全栈开发模式：

- **表示层**：React 18 + Tailwind CSS + Framer Motion。
- **逻辑层**：Next.js API Routes / Server Actions。
- **算法层**：通过 Node.js 子线程调度 `scipy` / `pymoo` 数学模块。
- **模型层**：本地 Ollama 服务接口（DeepSeek-R1 + Qwen3 异构模型集群）。
- **数据层**：ChromaDB（向量数据库）+ 本地结构化缓存。
